# 飞行器总体设计

使用本技能把 `$ARGUMENTS` 转换为有依据、可校核、可迭代的飞行器总体设计方案。默认用中文工作。所有关键数值必须标明来源：用户给定、本地知识库证据、计算得到、或工程假设。

## 首要动作

1. 将设计描述拆成需求矩阵：用途、载荷、航程/航时、速度/高度、起降、爬升、升限、过载/机动、动力/燃料、适航或战术约束、成本与维护、交付物。
2. 判断设计分支：民用运输机、通航/无人机、军用固定翼、运输/特种飞机、旋翼机、高速/高超声速飞行器。若分支会影响约束公式或参数先验，先说明假设。
3. 先检索本地 `RAG-data`，再做设计推理。使用现有检索器：

```bash
python "${CLAUDE_PROJECT_DIR}/.clawd/skills/aircraft-design-rag/scripts/search_rag.py" --data-dir "${CLAUDE_PROJECT_DIR}/RAG-data" --query "<检索词>" --format json --use-cache --top-k 8 --max-snippet-chars 700
```

如果 `python` 不可用，使用启动 Clawd 的 Python 或工作区可用 Python，但保持脚本与参数不变。

## 检索策略

不要只跑一个大查询。按设计环节运行 3 到 6 个短查询，并结合用户给出的机型、任务、速度、高度、载荷、动力形式扩展：

- `飞机总体设计 任务剖面 翼载 推重比 起飞重量 约束分析`
- `约束边界分析 推重比 翼载荷 起飞 着陆 爬升 盘旋`
- `民用飞机总体设计 初始参数估算 巡航马赫数 航程 爬升 起飞着陆`
- `军用飞机总体设计 任务剖面 过载 盘旋 爬升 推重比 翼载荷`
- `方案三视图 飞机几何参数 展弦比 后掠角 翼面积 机身长度`
- `重量平衡 控制 空机重量 燃油重量 起飞重量 重心`
- `发动机选型 推力 特性曲线 耗油率 总体设计`

优先使用这些主题文件：`飞机总体设计`、`大飞机总体设计`、`飞机设计手册_第4册_军用飞机总体设计`、`飞机设计手册_第5册_民用飞机总体设计`、`飞机设计手册_第6册_气动设计`、`飞机设计手册_第8册_重量平衡与控制`、相关动力装置手册。

若任务较完整，读取 `references/workflow.md`；其中有更详细的查询、约束、绘图和输出规范。

## 总体流程

参考用户给出的流程图，但按闭环工程流程组织：

1. 需求提取  
   提取总体设计约束、界限线图设计约束、巡航状态、爬升高度、最大巡航马赫数、其他设计输入。把自然语言要求转成可计算参数。

2. 历史型号与经验参数  
   形成数据库检索提示词，检索同类历史型号与教材案例。用它们给翼载荷、推重比、展弦比、后掠角、升阻比、最大升力系数、燃油系数、空重系数、发动机数量等赋先验范围。

3. 约束翻译与界限线准备  
   把起飞、着陆、爬升、巡航/最大速度、升限、加速、盘旋/过载、使用升限、最大马赫数等约束翻译为 W/S - T/W 或 W/P 平面上的曲线。输出每条曲线的数据点、适用公式、取值假设和可行侧。

4. V0 首轮总体参数估算  
   根据任务剖面、燃油/能量分数、空机重量关系、乘员和有效载荷估算起飞重量，得到初始翼载荷、推重比、翼面积、总推力/功率、燃油/能量重量。

5. 方案界限线图绘制  
   使用 `scripts/plot_constraint_boundary.py` 绘制并输出方案界限线图。图中必须包含各约束曲线、可行侧说明、初选设计点、必要的经验参考区间或历史型号点。

6. V1 布局综合  
   选择机翼、机身、尾翼、起落架、发动机/推进系统、燃油/载荷舱、增升装置和主要系统布置。输出三视图绘制所需参数：机身长/高/宽、翼展、翼面积、展弦比、后掠角、尖削比、根/尖弦、平均气动弦、尾翼面积与尾臂、发动机位置、起落架轮距/轴距、重心范围。

7. 分支校核  
   校核最大升力系数、起飞/巡航/跨声速约束、爬升率、使用升限、布局参数、燃油系数、空重因子、重心/稳定性、结构过载、降落过载、任务燃油/能量和跑道要求。

8. V2 闭合与迭代  
   用 V1 的气动、动力、重量和布局假设回算任务剖面与起飞重量。若起飞重量、翼载荷、推重比或关键几何变化超过约 5%，说明原因并建议再迭代；若接近或超过约 30%，判断初始设计点或布局可能不合理。

9. 输出与结束  
   输出推荐方案、参数表、界限线图文件路径、三视图参数、校核表、风险和下一步 CFD/风洞/MDAO/结构分析建议。

## 方案界限线图

当用户要求总体设计方案、约束分析或界限线图时，必须输出图。优先输出 SVG，因为无需额外依赖且可直接打开。

绘图脚本位置：

```bash
python "${CLAUDE_SKILL_DIR}/scripts/plot_constraint_boundary.py" --input "<约束图规格.json>" --output "<方案界限线图.svg>" --csv "<方案界限线图数据.csv>"
```

约束图规格 JSON 的基本结构：

```json
{
  "title": "方案界限线图",
  "x_label": "翼载荷 W/S (kg/m^2)",
  "y_label": "推重比 T/W",
  "x_range": [200, 800],
  "y_range": [0.2, 1.4],
  "constraints": [
    {
      "name": "起飞距离约束",
      "sense": "above",
      "points": [[250, 0.42], [350, 0.48], [450, 0.56], [550, 0.68]]
    },
    {
      "name": "着陆距离约束",
      "sense": "left",
      "x": 520
    }
  ],
  "design_points": [
    {"name": "推荐设计点", "x": 430, "y": 0.72}
  ],
  "regions": [
    {
      "name": "初选可行域",
      "points": [[300, 0.75], [500, 0.82], [500, 1.15], [300, 1.15]]
    }
  ]
}
```

`sense` 含义：`above` 表示曲线上方可行；`below` 表示曲线下方可行；`left` 表示曲线左侧可行；`right` 表示曲线右侧可行。若某条约束无法可靠数值化，仍需在报告中列出公式、缺失输入和建议取值，不能硬造曲线。

## 输出格式

完整总体设计任务按以下结构输出：

- `需求解析`：需求矩阵、假设、缺失输入。
- `依据检索`：本地文件路径、行号范围、每条证据支持的设计环节。
- `任务剖面`：各航段速度/高度/时间/距离/重量状态和备份燃油或能量逻辑。
- `约束翻译`：每条约束的公式、参数、可行侧和数据点。
- `方案界限线图`：SVG 文件路径、CSV 数据路径、推荐设计点。
- `总体参数`：起飞重量、空机重量、有效载荷、燃油/能量、翼面积、翼展、展弦比、推力/功率、CLmax、L/D、主要几何。
- `布局方案`：构型选择、动力布置、起落架、尾翼、燃油/载荷布置、三视图参数。
- `校核结果`：起飞、着陆、爬升、巡航/航程、升限、机动、燃油/能量、重心/稳定性、结构、运行约束。
- `迭代记录`：V0/V1/V2 的差异和是否需要下一轮。
- `风险与下一步`：主要风险、需要补充的数据、推荐仿真/试验/MDAO 工作。

若用户只要求子任务，输出相关子集，但仍需引用本地证据。

## 约束

- 不要脱离本地知识库直接凭记忆给结论。
- 不要编造精确参数；证据不足时给范围并标为工程假设。
- 公式输入不充分时保持符号化，列出缺失项。
- 区分民机、军机、无人机、旋翼机、高速/高超声速飞行器，不混用参数先验。
- 涉及武器载荷时，只讨论总体性能、载荷包线、重量和布置层面的工程约束；不提供武器使用、打击、规避或杀伤优化细节。
