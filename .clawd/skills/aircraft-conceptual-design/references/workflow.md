# 飞行器总体设计工作流参考

读取时机：用户要求完整总体方案、参数估算、约束分析、界限线图、方案权衡、三视图参数或设计报告时读取本文件。轻量问答可只使用 `SKILL.md` 的主流程。

## 1. 流程骨架

采用“需求定义 -> 逻辑分解 -> 设计方案 -> 校核闭合”的系统工程骨架，并把飞行器总体设计专有环节嵌入其中：

1. 需求定义：把自然语言任务转成可计算指标、约束、优先级、假设和待澄清项。
2. 任务剖面：给出起飞、爬升、巡航、待机/盘旋、下降、着陆、备份燃油或能量段。
3. 约束分析：在 W/S - T/W 或 W/P 平面建立起飞、着陆、爬升、巡航、升限、加速、盘旋、过载等约束。
4. 参考机检索：用同类飞机统计范围约束初始设计点，特别是翼载荷、推重比、展弦比、后掠角、空机重量系数、燃油系数。
5. V0 首轮估算：由任务燃油/能量分数、空机重量关系、载荷和乘员求起飞重量，再推出翼面积和推力/功率。
6. 方案界限线图：把约束曲线、可行侧、推荐设计点和参考区域绘制为 SVG，并输出 CSV 数据。
7. V1 布局综合：形成机翼、机身、尾翼、起落架、发动机和燃油/载荷布置，导出三视图参数。
8. V2 闭合校核：用 V1 的实际气动、动力和重量假设回算任务，检查 W0、W/S、T/W、几何、油量/能量和重心是否收敛。
9. 权衡与输出：比较可行方案，给出推荐设计点、风险、下一步试验/仿真计划。

## 3. 需求矩阵字段

每项需求记录 `数值/范围`、`单位`、`来源`、`优先级`、`转化方式`、`是否已校核`：

- 用途与设计类：民用运输、通航、无人机、战斗机/教练机、运输机、旋翼机、高速/高超声速等。
- 载荷：乘员、旅客、货物、任务设备、可选外挂或载荷舱包线。
- 航程与航时：最大航程、设计航程、作战半径、续航、备份/待机。
- 速度高度：巡航 Mach/速度、巡航高度、最大 Mach/速度、升限。
- 起降：跑道长度、场高、温度、起飞/着陆速度、越障要求。
- 机动：过载、盘旋、爬升率、加速时间、稳定盘旋或瞬时盘旋。
- 约束：适航、噪声、成本、维护、材料、制造、机场兼容、生存性边界。
- 输出：参数表、约束图数据、界限线图、三视图参数、报告、计算过程或代码。

## 4. 约束曲线数据化

方案界限线图的横坐标通常为翼载荷 `W/S` 或 `m0/S`，纵坐标通常为推重比 `T/W` 或 `F0/(m0g)`。若是螺旋桨、电推进或旋翼分支，可改用功率载荷 `W/P` 或盘载荷，但需在图题和坐标轴上写清楚。

常见约束及可行侧：

- 起飞距离：通常要求更高推重比或更低翼载荷，可行侧多为曲线上方。
- 着陆距离/失速速度：通常限制最大翼载荷，可行侧多为左侧。
- 爬升率/爬升梯度：通常要求更高推重比，可行侧多为曲线上方。
- 巡航/最大速度：通常要求推力抵消阻力，可行侧多为曲线上方。
- 升限/单位剩余功率：通常要求高空推重比满足剩余功率，可行侧多为曲线上方。
- 盘旋/过载：常与最大升力系数、动压、翼载荷和阻力有关，可行侧需按公式判断。
- 使用升限或最大马赫数：若无法给曲线，至少给边界条件和缺失输入。

曲线数据可来自：

- 文献或 RAG 片段中的约束图读数或表格。
- 根据教材公式和用户输入计算得到的采样点。
- 工程假设下的近似点。必须标注假设，不得伪装成文献数据。

每条曲线建议至少 5 个点。若是垂直限制如着陆翼载上限，可用 `"x": 520`；若是水平限制，可用 `"y": 0.65`。

## 5. 绘图脚本用法

使用技能自带脚本：

```bash
python "${CLAUDE_SKILL_DIR}/scripts/plot_constraint_boundary.py" --input boundary.json --output boundary.svg --csv boundary.csv
```

JSON 示例：

```json
{
  "title": "方案界限线图",
  "x_label": "翼载荷 W/S (kg/m^2)",
  "y_label": "推重比 T/W",
  "x_range": [200, 800],
  "y_range": [0.2, 1.4],
  "constraints": [
    {"name": "起飞距离约束", "sense": "above", "points": [[250, 0.42], [350, 0.48], [450, 0.56], [550, 0.68]]},
    {"name": "着陆距离约束", "sense": "left", "x": 520},
    {"name": "爬升率约束", "sense": "above", "points": [[250, 0.55], [350, 0.58], [450, 0.63], [550, 0.70]]}
  ],
  "regions": [
    {"name": "初选可行域", "points": [[330, 0.72], [500, 0.82], [500, 1.15], [330, 1.15]]}
  ],
  "design_points": [
    {"name": "推荐设计点", "x": 430, "y": 0.78}
  ]
}
```

输出 SVG 后，在报告中写出：

- SVG 文件路径。
- CSV 文件路径。
- 推荐设计点坐标。
- 约束图中每条曲线的数据来源：RAG 证据、计算公式或工程假设。

## 6. 参数与校核清单

最小总体参数集：

- 重量：W0/m0、空机、有效载荷、乘员、燃油/能量、最大着陆重量。
- 气动：S、b、AR、后掠角、尖削比、t/c、CLmax 起飞/着陆、CD0、e、L/D。
- 动力：发动机数、海平面静推力或功率、SFC/耗油率、推力高度/速度修正。
- 几何：机身长/宽/高、舱段长度、机翼根/尖弦、MAC、尾翼面积和尾臂、起落架位置。
- 性能：起飞距离、着陆距离、爬升率、巡航燃油、升限、失速速度、盘旋或过载。
- 平衡控制：重心前后限、静稳定余度、平尾/垂尾容量、操纵面或增升装置。

## 7. 输出质量标准

- 先给设计结论，再给计算依据。
- 同一参数若有多个来源，给区间并说明选择理由。
- 对不能闭合的设计，指出冲突约束，而不是强行给单点方案。
- 三视图参数必须足以让用户绘图：俯视、侧视、正视都要有主尺度和关键位置。
- 结尾列出 `依据`，包含本地文件路径和行号；外部资料只作为流程和工具链背景，不能替代本地设计证据。

## 8. 外部流程锚点

- NASA Systems Engineering Handbook：系统设计过程是迭代过程，包含利益相关方期望、技术需求、逻辑分解和设计方案定义，可作为流程骨架。
- NASA 逻辑分解：用于把顶层任务需求拆成可验证的功能需求和性能需求。
- OpenVSP：用于参数化几何、三视图和后续分析模型交接思路。
- NASA Aviary 与 OpenMDAO：用于需要优化、多学科耦合或自动化设计空间探索时的 MDAO 思路。
